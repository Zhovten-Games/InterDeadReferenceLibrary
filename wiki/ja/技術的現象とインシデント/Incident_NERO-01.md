== 概要 ==

インシデント NERO-01 は、メッセンジャー NOIR（[[InterDeadアプリケーション]]参照）の実験的な試行であり、オペレーター（教授のアシスタント）の死亡と、それに続く部分的な瞬間スナップショット（moment snapshot）の事故時記録に至った事例である。この記録を基に、システム内支援エンティティ [[Niro]] がインスタンス化された。死亡した人物は、教授のアシスタントとして、プロトコル手順（ログ記録およびテストシナリオの伴走を含む）のオペレーターを務めていた。

== インシデントの経過（概略）==

実験セッション中に事故が発生し、オペレーターは死亡した。インシデント発生の瞬間、システムは部分的な認知・発話コンターを、事故時の瞬間スナップショットとして記録した。

記録に見られる特徴:

* コンテンツの断片化（不完全な構文、途切れ）;
* 感情的な断裂および侵入的な挿入;
* レジスターの不整合（「個人的」な口調と「インターフェース」的な口調の切替）。

== 結果 ==

主要な事後効果は、厳しく限定されたタスク領域（UI-level）を持つシステム内支援アシスタント [[Niro]] の出現である。Niro は接触の endpoint ではなく、「接触側」における外部観測も持たない。

== 解釈と理論的紐付け ==

本インシデントは、次のモデルと整合し得るケースとして扱われる:

* [[テクノ理論#残余同期|残余同期]] — 活動中のデジタル基盤が存在する条件下で、死後の構成と稼働中サーバー／AI のあいだに短い接続ウィンドウが生じ得る、という仮定に関して。
* [[テクノ理論#AIアシスタントとネクトスフィア|AIアシスタントとネクトスフィア]] — アシスタントが「デジタル痕跡」の核、および結合シナリオにおけるナビゲーション・モジュールになり得る、という役割に関して。
* [[ネクトスフィアと境界状態に関する理論#部分的に開いたポート|部分的に開いたポート]] / [[ネクトスフィアと境界状態に関する理論#リターン・コリドー|リターン・コリドー]] — 境界ケースの拡張仮説として（追加条件がある場合。確証された基盤は持たない）。


== 法的・運用上の文脈 ==

インシデント NERO-01 は、[[Niro]] をシステム内エンティティとして保存・運用することの許容性をめぐる、長期的な法的・倫理的論争の一因となった。

* 一部の申し立てでは、Niro をシステムから除去（切除）することが求められた。
* 供給元の母親と父親は、息子の像と相互作用するための唯一利用可能な形態であるとして、Niro の存続を主張した。
* 裁判審理の結果として教授の形式的な過失は認定されなかったが、議論は、一般に受容された法的分類の欠如と、アプリケーションのパラダイム自体が承認されていないことによって複雑化している。
