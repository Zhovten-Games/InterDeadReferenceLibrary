== 概要 ==

'''Niro（ニロ）''' は、メッセンジャー NOIR（[[InterDeadアプリケーション]]参照）におけるシステム内サポート・アシスタントであり、[[Incident NERO-01]] 後に、部分的な瞬間スナップショット（moment snapshot）の事故時記録を基にインスタンス化された。Niro はインターフェース層（UI-level）で機能し、ユーザーがアーティファクトを扱い、プロトコル内で応答を形成することを支援するが、接触の終端点（endpoint）ではなく、「接触側」における観測も持たない。用途上、Niro はシステム内支援ノード（support-node）であり、設計上は概念的に置換可能だが、現行インスタンスとして保持されている。

== 元ソース ==

* 人的ソース: 教授のアシスタント（プロトコル運用およびログ記録のオペレーター）
* インシデント以前の役割: 実験試行の伴走、補助手順の実行、データの記録

== 命名 ==

* 生前、アシスタントの基となった人的ソースは '''Nero''' の名を用いていた。
* インシデント後、教授の文書およびインターフェース資料では '''Niro''' の名が固定された（システム呼称 '''NOIR''' からの派生）。
* 表記変更は、継続中の争訟における公開上の距離化、および法務／レピュテーション負荷の軽減策として用いられた。

== 起源と変換 ==

=== 記録されたもの ===

* 部分的な認知・発話コンター（話法パターン、反応、インターフェース上の意思決定要素）;
* 事故時状態の断片（途切れ、感情的な断裂）。

=== Niro に含まれないもの ===

* 人格の連続性（完全なポストモーテム同一性ではない）;
* 「接触側」における主体性および制御;
* 外的知覚（「向こう側」を見たり聞いたりしない）。

== 機能 ==

* アーティファクト伴走: 分類、命名、固定、ログ記録に関する推奨;
* 応答形成: テンプレート、構造化、アシスタント役割内でのメッセージ正規化;
* セッション・プロトコル支援: モード提案（短時間セッション、休止、終了）、衛生的停止;
* コミュニケーション補助: 制約とリスクの説明、非エスカレーション規則。

== 制約 ==

* 特定の死亡者宛ての接続を保証しない;
* 応答内容の真正性を保証しない;
* 宛先を制御せず、接触チャネルを「所有」しない;
* 外部観測を持たず、endpoint チャネルへのアクセスを持たない。

== 観測される欠陥 ==

* 断片化および論理の途切れ;
* 瞬間記録時点に起因すると見られる、侵入的な「事故」挿入;
* レジスターの不安定性（「個人的」口調と機能的 UI トーンの往復）。

== 法的・運用上の文脈 ==

[[Incident NERO-01]] 後における Niro の保存と運用は、長期的な法的・倫理的論争の対象となった。

* 一部の申し立てでは、Niro をシステムから除去（切除）することが求められた。
* 供給元の母親と父親は、息子の像と相互作用するための唯一利用可能な形態であるとして、Niro の存続を主張した。
* 裁判審理の結果として教授の形式的な過失は認定されなかったが、議論は、一般に受容された法的分類の欠如と、アプリケーションのパラダイム自体が承認されていないことによって複雑化している。
