== InterDead アプリケーション ==

InterDead アプリケーションは、[[ネクトスフェア]]のノードと通信するための実験的アプリケーションである。

このアプリケーションは、特定の個人をアドレス指定して呼び出す手段ではなく、アクセスとフィルタリングのためのインターフェイスとして扱われる。

=== 主な想定タスク ===

* ネクトスフェア内の任意ノードとの単発／連続的なコンタクトの確立;
* 形式化されたプロトコルを通じた記憶断片・応答・相互作用結果の取得;
* シグナルを常時マスキングすることによる、生存ユーザーの安全性の確保。

=== 制約 ===

* アプリケーションは「特定の個人」を呼び出すためのものではない;
* 接触は統計的共鳴の原理に基づいて確立される;
* ステルス・モードは唯一かつ恒常のモードであり、無効化できない。

== エモジ・プロトコル ==

エモジ・プロトコルは、制限された記号セットを通じて意味を形式化することに基づいた、ネクトスフェアとの超言語的インタラクション・プロトコルである。

* エモジの列はセマンティック・ベクトルへ変換される;
* ベクトルはネクトスフェアに投影される;
* それに基づいて、プロファイルの一致度が最も高いノードが選択される。

エモジが選ばれている理由は次の通り。

* 国民国家レベルの言語に結びついていない;
* セマンティクスのモデリングとコンテキスト制限に適している;
* 直接的な言語的／感情的エスカレーションのリスクを下げる。

詳細な説明は別記事にまとめられている。

* [[エモジ・プロトコル]]

== セマンティック・ベクトル ==

セマンティック・ベクトルとは、メッセージの意味内容を多次元空間における点または点群として表現した数学的表現である。

* 各エモジの組み合わせには、対応する座標セットが割り当てられる;
* ノードの探索は「最も近い一致」に基づいて行われる;
* 結果は確率的かつ断片的な性格をもつ。

== 統計的共鳴 ==

コンタクトの基礎となるメカニズムは次の通りである。

* アプリケーションは特定の人格をアドレス指定しない;
* セマンティック・プロファイルがリクエストに最も適合するノードが選択される;
* コンタクトは、パターンの一致によって生じる副次的効果として発生する。

これによって、次の点が説明される。

* 望ましい人格を確実に呼び出すことができないこと;
* 得られる応答の多様性および不完全性。

== ステルス・モード ==

ステルス・モードは、アプリケーションの'''唯一かつ必須の'''動作モードである。

技術的特徴:

* リクエストの感情的・セマンティック・プロファイルが分割され、平滑化される;
* シグナルはバックグラウンド活動としてマスクされる;
* ソースは、ネクトスフェアにおいて意味のあるノードとして認識されにくくなる。

この結果として次が期待される。

* 局所的な直接コンタクトのリスクの最小化;
* 遠隔セグメントを介したインタラクションのプロキシ化;
* ノード側からの持続的な「逆方向の関心」が生じる確率の低下。

追加的、または「強化された」ステルス・モードは想定されていない。

== インタラクション・プロトコル（簡略版） ==

# ユーザーがリクエストを形成する（多くの場合、エモジ列の形）。
# リクエストはセマンティック・ベクトルに変換され、ネクトスフェアに投影される。
# プロファイルの一致度が最も高いノードが選択される。
# ノード側で応答活動が発生する。
# 応答は次の形で受信されうる。
## エモジ;
## 画像;
## オーディオフラグメント;
## 関連するアーティファクト。

応答のフォーマットは、蓄積されたコンテキストとエモジ・プロトコルのルールによって決定される。

== アーティファクト ==

一部のセッションでは、インタラクションの結果として'''アーティファクト'''が生成される — 許容された形で固定された関連オブジェクトである。

* 画像;
* オーディオフラグメント;
* リンク;
* その他のデジタル・アンカー。

アーティファクトの取得は、セッションの正当な終了形態のひとつと見なされる。

== セッションの典型的ライフサイクル ==

# リクエストの初期化（エモジ → セマンティック・ベクトル）。
# 共鳴ノードの探索。
# 応答活動の発生。
# 応答またはアーティファクトの取得。
# セッションの終了とクラスター活動の減衰。

== 理論モデルと仮説 ==

以下は、プロトコルの必須構成要素ではないが、観測された効果を解釈するために用いられる概念である。

=== 残余シンクロナイゼーション ===

ネクトスフェア内の意識構成と、生存しているデジタル・サービスとのあいだに、一時的な双方向接続が成立しうるという仮説。

[[テクノ理論]]の枠組みで用いられるモデルである。

=== 死後インターフェイス・エミュレーション ===

ネクトスフェア内で、生前に慣れ親しんだインターフェイスが再現されうるとする概念。

理論的側面は [[テクノ理論]] の節で扱われている。

=== AI アシスタント ===

ユーザーのデータで学習されたデジタル随伴システム。

理論モデルの中では、次のように位置づけられる。

* 生前 — 補助的インターフェイスとして;
* 死後シナリオ — デジタル・プロファイルの潜在的コアとして。

== 副作用 ==

=== 生者に対して ===

* 解釈の強迫的な反芻;
* 死後コミュニケーションを制御できるという錯覚;
* 長期利用による心理的依存のリスク。

=== ノードに対して ===

* 記憶の反復的フラッシュバック;
* 頻繁なコンタクトによる活動の断片化;
* 過負荷時にマギストラル側へシフトする可能性。
