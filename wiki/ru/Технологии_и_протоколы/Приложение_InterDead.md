== Приложение InterDead ==

Экспериментальное приложение для коммуникации с узлами [[Нектосфера|нектоcферы]].

<code>'''Терминология:''' в документах профессора и части внутренних материалов приложение обозначается как '''NOIR''' (Nectosphere-Oriented Interface Relay) — это рабочее имя мессенджера/системы. В данной вики далее используется название «Приложение InterDead».</code>

Приложение рассматривается как интерфейс доступа и фильтрации, а не как средство адресного вызова конкретных личностей.

=== Основные заявленные задачи ===

* установление разовых или серийных контактов с произвольными узлами нектоcферы;
* получение фрагментов памяти, откликов и результатов взаимодействия через формализованные протоколы;
* обеспечение безопасности живого пользователя за счёт постоянной маскировки сигнала.

=== Ограничения ===

* приложение не предназначено для вызова «конкретного человека»;
* контакт устанавливается по принципу статистического резонанса;
* режим скрытности является единственным, постоянным и неотключаемым.

== Эмодзи-протокол ==

Эмодзи-протокол — надъязыковой протокол взаимодействия с нектоcферой, основанный на формализации смысла через ограниченный набор знаков.

* последовательность эмодзи преобразуется в семантический вектор;
* вектор проецируется в нектоcферу;
* по нему выбирается узел с наибольшим совпадением профиля.

Причины выбора эмодзи:

* они не привязаны к национальному языку;
* они удобны для семантического моделирования и ограничения контекста;
* они снижают риск прямой языковой и эмоциональной эскалации.

== Семантический вектор ==

Семантический вектор — математическое представление смысла сообщения в виде точки или облака в многомерном пространстве.

* каждой комбинации эмодзи соответствует набор координат;
* поиск узла осуществляется по принципу «ближайшего совпадения»;
* результат носит вероятностный и фрагментарный характер.

== Статистический резонанс ==

Механизм, лежащий в основе контакта:

* приложение не адресуется к конкретной личности;
* выбирается узел, наиболее соответствующий семантическому профилю запроса;
* контакт возникает как побочный эффект совпадения паттернов.

Это объясняет:

* невозможность гарантированного вызова желаемой личности;
* вариативность и неполноту получаемых откликов.

== Режим скрытности ==

Режим скрытности является '''единственным и обязательным''' режимом работы приложения.

Технические особенности:

* эмоционально-семантический профиль запросов фрагментируется и сглаживается;
* сигнал маскируется под фоновую активность;
* источник не воспринимается нектоcферой как значимый узел.

Следствия:

* минимизация риска локального прямого контакта;
* проксирование взаимодействий через удалённые сегменты;
* снижение вероятности устойчивого «обратного интереса» со стороны узлов.

Дополнительных или «усиленных» режимов скрытности не предусмотрено.

== Протокол взаимодействия (упрощённо) ==

# пользователь формирует запрос (чаще всего в форме эмодзи-последовательности).
# запрос преобразуется в семантический вектор и проецируется в нектоcферу.
# выбирается узел с наибольшим совпадением профиля.
# на стороне узла возникает ответная активность.
# ответ может быть получен:
## в виде эмодзи;
## в виде изображения;
## в виде аудиофрагмента;
## в виде связанного артефакта.

Формат ответа определяется накопленным контекстом и правилами эмодзи-протокола.

== Артефакты ==

В ряде сессий результатом взаимодействия становится артефакт — связанный объект, зафиксированный в допустимой форме:

* изображение;
* аудиофрагмент;
* ссылка;
* иной цифровой якорь.

Получение артефакта считается корректным возможным завершением сессии.

== Типичный жизненный цикл сессии ==

# инициализация запроса (эмодзи → семантический вектор).
# поиск резонансного узла.
# возникновение ответной активности.
# (Опционально, по ситуации) если запрос подразумевает съёмку, но допустимость передачи изображения неочевидна, может быть введён промежуточный шаг согласия/уточнения допустимости.
# получение отклика или артефакта.
# завершение сессии и спад активности кластера.

== Теоретические модели и гипотезы ==

Ниже перечислены концепции, не являющиеся обязательной частью протокола, но используемые для интерпретации наблюдаемых эффектов.

=== Резидуальная синхронизация ===

Гипотеза временной двусторонней связи между конфигурацией сознания в нектоcфере и живыми цифровыми сервисами.

Рассматривается как модель в рамках [[Техно-теории]].

=== Посмертная эмуляция интерфейсов ===

Концепция, описывающая возможное воспроизведение в нектоcфере интерфейсов, привычных для личности при жизни.

Теоретические аспекты изложены в разделе [[Техно-теории]].

=== AI-ассистент ===

В теоретических моделях рассматривается:

* при жизни — как вспомогательный интерфейс;
* в посмертных сценариях — как потенциальное ядро цифрового профиля.

В рамках мессенджера реализован внутрисистемный ассистент поддержки [[Niro]], инстанцированный после [[Инцидент NERO-01]]. Ниро функционирует на уровне интерфейса: сопровождает работу с артефактами и помогает структурировать сообщения и ход сессии; при этом он не является контактной конечной точкой (endpoint) и не обладает наблюдением на «контактной стороне».

== Побочные эффекты ==

=== Для живых ===

* навязчивый анализ интерпретаций;
* иллюзия контроля над посмертной коммуникацией;
* риск психологической зависимости при длительном использовании.

=== Для узлов ===

* повторяющиеся вспышки воспоминаний;
* фрагментация активности при частых контактах;
* возможное смещение к магистралям при перегрузке.
