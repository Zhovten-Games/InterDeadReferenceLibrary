== Приложение InterDead ==
Экспериментальное приложение для общения с мёртвыми.

Основные заявленные задачи:

* установление разового/серийного контакта с произвольными узлами [[Нектосфера|нектосферы]];
* получение фрагментов их воспоминаний через эмодзи-протокол;
* защита живых пользователей от прямого контакта с локальными духами (режим скрытности).

Ограничения:

* не предназначено для вызова «конкретного человека»;
* работает по принципу статистического резонанса (см. ниже).

=== Эмодзи-протокол ===
Надъязыковой протокол обмена с нектосферой на основе эмодзи.

Описание:

* последовательность эмодзи → преобразуется в семантический вектор;
* вектор проецируется в нектосферу;
* по нему выбирается кластер с максимальным совпадением в «облаке смыслов».

Причины выбора эмодзи:

* они не привязаны к национальному языку;
* они хорошо подходят для семантического моделирования как наборы смысловых признаков.

=== Семантический вектор ===
Математическое представление смысла сообщения в виде точки/облака в многомерном пространстве.

Использование:

* каждой комбинации эмодзи → соответствует набор координат;
* поиск [[Узел нектосферы|узла нектосферы]] идёт по принципу «найти ближайшую точку/кластер».

=== Статистический резонанс ===
Механизм установления связи с мёртвыми.

Суть:

* приложение не обращается к конкретному умершему;
* алгоритм ищет [[Узел нектосферы|узел]] с максимальным совпадением семантического профиля;
* связь устанавливается с тем, чей паттерн «лучше всего подходит» запросу.

Это объясняет:

* невозможность гарантированного вызова «нужной личности»;
* случайный характер контактов.

=== Режим скрытности ===
Режим маскировки сигнала пользователя в нектосфере. Является ''базовым и обязательным'' режимом для потребительских сборок приложения.

Техническое поведение:

* эмоционально-семантический профиль запросов «дробится» и сглаживается;
* сигнал приводится к виду фонового шума;
* маршрутизаторы нектосферы не воспринимают источник как значимый узел.

Последствия:

* локальные духи не регистрируют источник как объект интереса;
* запросы проксируются в удалённые сегменты;
* риск прямого контакта с «обитателями места» снижается.

Правовой и регуляторный статус:

* после ряда инцидентов и судебных исков отключение режима скрытности в пользовательских (потребительских) версиях приложения запрещено;
* изменение режима возможно только через внутренние механизмы разработки/отладки и не документировано в публичной пользовательской документации.

=== Режим экспозиции ===
Режим, противоположный режиму скрытности.

Функции:

* резкое увеличение [[Яркость узла|яркости узла]];
* превращение его в «маяк» для нектосферных процессов;
* создание условий для формирования направленного канала (например, предполагаемого [[Коридор возврата|коридора возврата]]) к соответствующему порту живого тела или другому целевому интерфейсу.

Риски:

* узел становится заметным и привлекательным для других мёртвых;
* возрастает вероятность «погони» и попыток перехвата канала.

Доступность:

* режим недоступен в обычных пользовательских сборках;
* включение возможно только в режиме разработчика / исследовательских конфигурациях:
** через отдельные диагностические интерфейсы;
** с многошаговым подтверждением и жёсткими юридическими дисклеймерами;
* в спецификациях протокола описывается как экспериментальный и высокорисковый режим (см. также раздел [[Теории о нектосфере и пограничных состояниях]]).

=== Резидуальная синхронизация ===
Термин, обозначающий гипотетическую временную двустороннюю связь между конфигурацией сознания в нектосфере и живыми цифровыми сервисами.

Подробно рассматривается как модель в разделе [[Техно-теории]].

=== Посмертная эмуляция интерфейсов ===
Концепция, описывающая возможное воспроизведение в нектосфере интерфейсов, привычных для личности.

Теоретические аспекты и предполагаемый механизм изложены в разделе [[Техно-теории]].

=== AI-ассистент ===
Цифровая система поддержки и сопровождения пользователя, обученная на его данных (история действий, запросов, предпочтений).

Роли в рамках модели:

* при жизни — классический ассистент/органайзер, интегрированный с приложением InterDead и/или другими сервисами;
* в посмертных сценариях — рассматривается как потенциальное ядро цифрового профиля, способное участвовать в расширенных протоколах взаимодействия с нектосферой (см. раздел [[Техно-теории]]).

== Документация по приложению InterDead (краткая техническая спецификация) ==

=== Назначение ===
* Установление связи с произвольными узлами [[Нектосфера|нектосферы]].
* Получение фрагментов воспоминаний через эмодзи-протокол.
* Обеспечение базовой безопасности пользователя через режим скрытности.

=== Архитектура ===
Компоненты:

* '''Клиент''' (на стороне пользователя):
** сбор эмодзи-запроса;
** первичная семантическая обработка;
** диалоговый интерфейс.

* '''Сервер доступа к нектосфере''':
** построение семантического вектора;
** поиск резонансного кластера в нектосфере;
** поддержание коммуникационного канала.

* '''Модуль защиты (скрытность/экспозиция)''':
** управление яркостью узла;
** маскировка или экспонирование в нектосфере.

При необходимости архитектура может быть расширена внешними AI-сервисами через API.

=== Режимы работы ===

==== Стандартный режим ====
* режим скрытности активен по умолчанию;
* в пользовательских сборках отключение скрытности не предусмотрено;
* одиночные сессии связи;
* яркость узла минимальна.

==== Режим скрытности (усиленный) ====
* дополнительное подавление сигнала;
* рекомендуется для групповых выездов в «проклятые места»;
* все контакты — только через удалённые сегменты.

==== Режим экспозиции ====
* используется в исключительных сценариях и только в сборках для разработчиков/исследований;
* яркость узла резко увеличивается;
* создаются условия для формирования направленных каналов связи при наличии порта;
* недоступен в пользовательских сборках.

=== Протокол взаимодействия (упрощённо) ===
# Пользователь формирует запрос в эмодзи.
# Клиент строит семантический вектор и отправляет его на сервер.
# Сервер ищет в нектосфере ближайший по профилю узел (статистический резонанс).
# На стороне узла возникает навязчивая мысль / вспышка активности.
# Ответ кодируется в виде образов, которые клиент интерпретирует и показывает пользователю (обычно через новые эмодзи и/или текстовые подсказки).

=== Ограничения ===
* Нет прямой адресации по личности («вызвать конкретного умершего» невозможно).
* Высокая доля шумовых и фрагментарных ответов.
* При длительных сессиях растёт риск ментальной дестабилизации пользователя (навязчивость интерпретации эмодзи).

=== Побочные эффекты для живых ===
* навязчивый поиск «правильной» интерпретации эмодзи;
* формирование устойчивых иллюзий контроля над посмертной коммуникацией;
* риск зависимости от приложения как единственного способа переживания потери.

=== Побочные эффекты для мёртвых ===
* навязчивые мысли и повторяющиеся вспышки воспоминаний;
* усиление фрагментации личности при постоянных неудачных сессиях;
* возможный сдвиг узла в сторону магистралей из-за повышенной нагрузки.

=== Типичный жизненный цикл сессии ===
# Инициализация запроса (эмодзи → семантический вектор).
# Поиск резонансного узла в нектосфере.
# Навязчивая мысль на стороне мёртвого.
# Ответ → интерпретация → новая комбинация эмодзи.
# Завершение сессии, спад активности в соответствующем кластере.
