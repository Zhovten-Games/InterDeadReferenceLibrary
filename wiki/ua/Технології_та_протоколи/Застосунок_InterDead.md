== Застосунок InterDead ==

Експериментальний застосунок для комунікації з вузлами [[Нектосфера|некстосфери]].

Застосунок розглядається як інтерфейс доступу та фільтрації, а не як засіб адресного виклику конкретних особистостей.

=== Основні заявлені завдання ===

* встановлення разових або серійних контактів із довільними вузлами некстосфери;
* отримання фрагментів пам’яті, відгуків і результатів взаємодії через формалізовані протоколи;
* забезпечення безпеки живого користувача за рахунок постійного маскування сигналу.

=== Обмеження ===

* застосунок не призначений для виклику «конкретної людини»;
* контакт встановлюється за принципом статистичного резонансу;
* режим прихованості є єдиним, постійним і таким, що не вимикається.

== Емодзі-протокол ==

Емодзі-протокол — надмовний протокол взаємодії з некстосферою, заснований на формалізації сенсу через обмежений набір знаків.

* послідовність емодзі перетворюється на семантичний вектор;
* вектор проєктується в некстосферу;
* за ним обирається вузол із найбільшим збігом профілю.

Причини вибору емодзі:

* вони не прив’язані до національної мови;
* вони зручні для семантичного моделювання та обмеження контексту;
* вони знижують ризик прямої мовної та емоційної ескалації.

== Семантичний вектор ==

Семантичний вектор — математичне представлення сенсу повідомлення у вигляді точки або хмари в багатовимірному просторі.

* кожній комбінації емодзі відповідає набір координат;
* пошук вузла здійснюється за принципом «найближчого збігу»;
* результат має імовірнісний і фрагментарний характер.

== Статистичний резонанс ==

Механізм, що лежить в основі контакту:

* застосунок не адресує запит конкретній особистості;
* обирається вузол, який найбільше відповідає семантичному профілю запиту;
* контакт виникає як побічний ефект збігу патернів.

Це пояснює:

* неможливість гарантованого виклику бажаної особистості;
* варіативність і неповноту отриманих відгуків.

== Режим прихованості ==

Режим прихованості є '''єдиним і обов’язковим''' режимом роботи застосунку.

Технічні особливості:

* емоційно-семантичний профіль запитів фрагментується й згладжується;
* сигнал маскується під фонову активність;
* джерело не сприймається некстосферою як значущий вузол.

Наслідки:

* мінімізація ризику локального прямого контакту;
* проксіювання взаємодій через віддалені сегменти;
* зниження ймовірності стійкого «зворотного інтересу» з боку вузлів.

Додаткових або «посилених» режимів прихованості не передбачено.

== Протокол взаємодії (спрощено) ==

# Користувач формує запит (найчастіше у формі емодзі-послідовності).
# Запит перетворюється на семантичний вектор і проєктується в некстосферу.
# Обирається вузол із найбільшим збігом профілю.
# На боці вузла виникає відповідна активність.
# Відповідь може бути отримана:
## у вигляді емодзі;
## у вигляді зображення;
## у вигляді аудіофрагмента;
## у вигляді пов’язаного артефакту.

Формат відповіді визначається накопиченим контекстом і правилами емодзі-протоколу.

== Артефакти ==

У низці сесій результатом взаємодії стає '''артефакт''' — пов’язаний об’єкт, зафіксований у допустимій формі:

* зображення;
* аудіофрагмент;
* посилання;
* інший цифровий якір.

Отримання артефакту вважається коректним можливим завершенням сесії.

== Типовий життєвий цикл сесії ==

# Ініціалізація запиту (емодзі → семантичний вектор).
# Пошук резонансного вузла.
# Виникнення відповідної активності.
# Отримання відгуку або артефакту.
# Завершення сесії та спад активності кластера.

== Теоретичні моделі та гіпотези ==

Нижче перелічено концепції, що не є обов’язковою частиною протоколу, але використовуються для інтерпретації спостережуваних ефектів.

=== Резидуальна синхронізація ===

Гіпотеза тимчасового двостороннього зв’язку між конфігурацією свідомості в некстосфері та живими цифровими сервісами.

Розглядається як модель у межах [[Техно-теорії]].

=== Посмертна емуляція інтерфейсів ===

Концепція, що описує можливе відтворення в некстосфері інтерфейсів, звичних для особистості за життя.

Теоретичні аспекти викладені в розділі [[Техно-теорії]].

=== AI-асистент ===

Цифрова система супроводу користувача, навчена на його даних.

У теоретичних моделях розглядається:

* за життя — як допоміжний інтерфейс;
* у посмертних сценаріях — як потенційне ядро цифрового профілю.

== Побічні ефекти ==

=== Для живих ===

* нав’язливий аналіз інтерпретацій;
* ілюзія контролю над посмертною комунікацією;
* ризик психологічної залежності при тривалому використанні.

=== Для вузлів ===

* повторювані спалахи спогадів;
* фрагментація активності при частих контактах;
* можливе зміщення до магістралей при перевантаженні.
