# Licensing

This repository applies the shared repository licensing policy published at:

- Policy source: `https://github.com/FOP-Oksana-Dubinetska/repository-licensing-policy`
- Git submodule path: `.licensing-policy/`
- Pinned policy version: `v1.0.0 (6e4c2627717c079827ed4aa9044a5346b3ea3ddb)`

The policy submodule is incorporated by reference. Its presence does not automatically license this repository: this file declares the licenses that apply to this repository.

## Default licensing map

Unless a more specific third-party notice, subtree notice, file-level notice, or project-specific override applies:

| Scope | Default license |
|---|---|
| Original research notes, primers, public reference documents, publication sources, Markdown documentation, and original non-brand editorial materials | `CC-BY-SA-4.0` |
| Original source code, scripts, tests, examples, reusable schemas, reusable templates, pipeline helpers, and build helpers | `MIT` |
| InterDead names, Zhovten Games names, project names, logos, trademarks, brand assets, visual identity elements, setting identity, worldbuilding identity, and fictional proper names | No license granted; all rights reserved |
| Third-party materials, quotations, external references, linked materials, imported assets, and upstream examples | Their original upstream licenses and notices |

## Project-specific clarification

This repository is a public reference library for the InterDead project.

The licenses above allow reuse of eligible original research and technical materials under the stated terms. They do not grant permission to reuse the InterDead setting, brand identity, fictional universe, project identity, logos, names, or other reserved identity materials as a competing product, derivative brand, or standalone fictional property.

In practical terms:

- research notes and documentation may be reused under `CC-BY-SA-4.0`, unless a more specific notice applies;
- code and reusable technical helpers may be reused under `MIT`, unless a more specific notice applies;
- the InterDead identity, Zhovten Games identity, logos, trademarks, fictional proper names, and brand/world identity are not licensed for reuse;
- third-party materials remain under their original upstream terms.

## Project-specific overrides

See `LICENSING-OVERRIDES.md` if that file exists.

If `LICENSING-OVERRIDES.md` does not exist, this repository declares no additional project-specific overrides beyond the clarification in this file.

## Precedence

Where multiple notices apply, use the following order:

1. mandatory third-party upstream terms and notices;
2. file-level notices and SPDX identifiers;
3. subtree-specific `LICENSE`, `LICENSE.md`, or `NOTICE` files;
4. `LICENSING-OVERRIDES.md`;
5. the project-specific clarification in this file;
6. the default licensing map above.

A project-specific override may clarify or replace a default only for material owned by the repository rights holder. It cannot weaken or replace third-party upstream obligations.
